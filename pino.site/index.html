Syntactic Elegance

Clean structures, optional commas, zero boilerplate. Discover how Pino reads.

Constants vs. Mutables

Distinguish cleanly between constant and mutable variables. Constant mutability checks are evaluated at parse-time to guarantee runtime immutability.

  • Use val for immutable properties.
  • Use var for values that can change.
  • Parser enforces compile-time warnings against reassignment.
# Define a constant value
val gravity = 9.81

# Define a mutable tracker
var velocity = 0
velocity = velocity + 10 # Reassignment is allowed

First-Class Interpolation

Interpolate variables directly inside double quotes using a simple $ symbol, or write full evaluations inside $(expr) tags. Interpolated blocks desegment into addition trees automatically.

  • Variable injection via $varName.
  • Expression interpolation via $(expression).
  • No complex compiler overrides needed; parses directly as concatenated string expressions.
val name = "Lenore"
val age = 28
println("Developer $name is $age years old.")
println("Next year: $(age + 1)")

Higher-Order Closures

Functions support optional parameter lists and zero commas. Pino lambdas generate local scopes carrying references to their parent environments, supporting closures.

  • Declare functions using fn.
  • Return lambdas dynamically to build functional closures.
  • Parameters do not require comma separators for faster layout.
# Higher-order function returning a multiplier closure
fn multiplier(factor int) {
  return fn (val int) {
    return val * factor
  }
}

val double_it = multiplier(2)
println(double_it(5)) # Prints 10

Structs & Nested Methods

Declare custom structured data with attributes and methods. Access members and invoke methods using the colon (:) operator, modifying attributes in-place.

  • Access fields using instance:field.
  • Invoke methods using instance:method().
  • In-place assignment supported (e.g. instance:field += value).
struct Point {
  x int
  y int
  fn move(dx int dy int) {
    x += dx
    y += dy
  }
}

var p = Point { x: 5, y: 10 }
p:move(2, 5)
println("Position: $(p:x), $(p:y)") # 7, 15

Unified Loop Structures

Looping is simplified down to a single keyword: for. Pino supports infinite loops, fixed integer ranges, and iterator collections.

  • Infinite loops with for { ... }.
  • Range iterations with for index in 10 { ... }.
  • Collection parsing with for item in list { ... }.
# Iterating a range
for time in 5 {
  println("Step $time")
}

# Iterating a list
val items = ["A", "B", "C"]
for it in items {
  println(it)
}

Match-When Switches

Perform precise checks with standard if-else statement logic or implement high-performance, readable case switches via the match-when block.

  • Omit parenthesis in condition checks.
  • Supports multi-value matching: when "Mars", "Venus".
  • Default fallthrough routing using else.
val action = "Compile"
match action {
  when "Compile" {
    println("Building executable...")
  }
  when "Clean", "Reset" {
    println("Cleaning artifacts...")
  }
  else {
    println("Unknown command.")
  }
}

Run Pino Locally

Get the standalone C# interpreter running on your local terminal in three simple steps.

1

Build compiler CLI

Clone the repository and build the C# compiler. Ensure you have the .NET 10 SDK installed on your system.

cd pino-csharp && dotnet build
2

Run your first script

Execute your pino source code file using the run command. Or use the watch command to enable hot-module reloading.

dotnet run run main.pino
